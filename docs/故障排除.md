# 故障排除

## 一、构建阶段

### 1. CMake 找不到 Visual Studio

**现象**：`could not find any instance of Visual Studio`。

**处理**：

- 安装 Visual Studio 并勾选 C++ 桌面开发  
- 使用本机已安装的生成器，例如 `"Visual Studio 17 2022"` 或 `"Visual Studio 18 2026"`  
- 切换生成器时删除整个 `build` 目录后重新配置  

### 2. PortAudio 配置报 CMake 策略错误

**现象**：`Compatibility with CMake < 3.5 has been removed`。

**处理**：配置时增加：

```powershell
"-DCMAKE_POLICY_VERSION_MINIMUM=3.5"
```

### 3. ASIOSDK_ROOT_DIR not found

**现象**：配置失败，提示找不到 `common/asio.h`。

**处理**：

- `ASIOSDK_ROOT_DIR` 必须指向 **ASIOSDK** 目录本身  
- 正确示例：`.../ASIO-SDK_2.3.4_2025-10-15/ASIOSDK`  
- 错误示例：`.../ASIO-SDK_2.3.4_2025-10-15`（少一层 ASIOSDK）  

### 4. MFC / WinMain 链接错误

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| `#error: Building MFC application with /MD requires MFC shared dll` | 未定义 `_AFXDLL` | 确认 `CMakeLists.txt` 中有 `_AFXDLL` 与 `UseOfMfc=Dynamic` |
| `unresolved external symbol WinMain` | 未链接 MFC 入口库 | 安装 MFC 组件；设置 `VS_GLOBAL_UseOfMfc` 为 `Dynamic` |
| 大量 `Pa_*` / `PaUtil_*` 未解析 | PortAudio 的 `.c` 未编译 | 确认 `project(... LANGUAGES C CXX)` |

### 5. CRT 运行库不匹配（LNK2038）

**现象**：`RuntimeLibrary` 不匹配 `MD` vs `MT`。

**处理**：本工程应用与 PortAudio 子项目均使用 **动态 CRT（/MD）**；勿对应用单独设置 `/MT` 的 `MSVC_RUNTIME_LIBRARY`，除非同时修改 PortAudio 的 `PA_DLL_LINK_WITH_STATIC_RUNTIME`。

## 二、运行时 — 设备与初始化

### 1. 日志：PortAudio init failed

**可能原因**：

- 系统无 ASIO 驱动  
- PortAudio 编译未包含 ASIO（检查 `PA_USE_ASIO`）  

**处理**：安装 Dante Virtual Soundcard 或其它 ASIO 驱动；用官方 ASIO 测试程序确认驱动正常。

### 2. 日志：No ASIO output devices found

**可能原因**：仅有 ASIO 输入设备、驱动未启动、驱动仅注册输入端点。

**处理**：

- 在 Windows 声音设置或 Dante 软件中确认 Virtual Soundcard 已启用  
- 重启驱动或重新插拔虚拟声卡服务  

### 3. 设备列表无 Dante / 未默认选中

**说明**：默认选中逻辑为名称包含 `Dante Virtual Soundcard`；名称略有差异时需手动在下拉框选择。

**处理**：查看日志或第三方工具（如 PortAudio 的 `pa_devs` 示例）中的实际设备名字符串，必要时可修改 `DeviceEnumerator.cpp` 中的匹配关键字。

## 三、运行时 — 播放

### 1. WAV load failed

| 日志关键词 | 原因 | 处理 |
|------------|------|------|
| Cannot open WAV file | 路径错误或无权限 | 检查路径、避免特殊字符、用 Browse 重选 |
| Not a RIFF/WAVE file | 非标准 WAV | 用 Audacity 等导出为标准 PCM WAV |
| Unsupported WAV format | 压缩或扩展格式 | 转为 PCM 或 32-bit float |
| Only mono or stereo | 多声道文件 | 混缩为单声道或立体声 |
| Missing fmt or data chunk | 文件损坏 | 重新生成文件 |

### 2. Playback start failed: Pa_OpenStream ...

**常见原因**：

| 原因 | 处理 |
|------|------|
| 采样率不被驱动支持 | ASIO 控制面板设为与 WAV 相同（推荐 48000） |
| 缓冲区大小非法 | 换用列表中带 `(preferred)` 的项或其它合法帧数 |
| 设备被占用 | 关闭 Reaper、DAW 等独占 ASIO 的程序 |
| 输出通道索引越界 | 降低 Output Channel，确保 &lt; `maxOutputChannels` |

**查阅错误详情**：日志中附带 `Pa_GetErrorText` 英文描述，可对照 [PortAudio 错误码文档](http://www.portaudio.com/docs/v19-doxydocs/portaudio_8h.html)。

### 3. 播放成功但 Dante 无电平

**检查清单**：

1. Output Channel 是否为 Dante 路由中实际监听的通道  
2. Dante Controller 中 TX 路由是否包含该通道  
3. Virtual Soundcard 通道是否与 Controller 配置一致  
4. 系统音量 / 静音（部分驱动不受影响，但需排除）  
5. LTC 文件是否为静音或极低电平  

### 4. 播放到一半卡顿或爆音

**可能原因**：缓冲区过小、USB/网络 Dante 负载过高、非实时线程干扰。

**处理**：增大 Buffer (frames)；关闭其它高负载 ASIO 客户端；使用 Release 构建测试。

### 5. 循环播放不生效

确认已勾选 **Loop playback**；若仍异常，查看是否在播完前手动点了 Stop，或日志是否出现 `Playback finished`（表示未勾选循环）。

## 四、调试建议

1. **先用立体声测试音**（非 LTC）确认通道有输出，再换 LTC 单声道文件。  
2. **固定 48000 Hz / 512 帧缓冲** 作为基线，再尝试其它参数。  
3. 需要完整设备参数时，可编译 PortAudio 自带的 `pa_devs` 示例对比通道数与默认采样率。  
4. 播放失败时优先打开 **ASIO Control Panel** 查看驱动报告的错误或采样率锁定状态。  

---

## 五、获取帮助时应提供的信息

- Windows 版本、Visual Studio / CMake 版本  
- Dante Virtual Soundcard 版本  
- 日志区完整复制（含 Warning 与 `Pa_GetErrorText`）  
- WAV 参数：采样率、声道、位深、时长  
- 所选设备名、Output Channel、Buffer frames
