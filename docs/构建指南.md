# 构建指南

## 1. 环境准备

### 操作系统

- Windows 10 或更高版本（x64）

### Visual Studio

安装 **「使用 C++ 的桌面开发」** 工作负载，并勾选：

- **MFC 和 ATL 支持**（共享 MFC DLL，非静态 MFC）

支持 Visual Studio 2022 或 2026（本仓库已在 VS 2026 / v145 工具集下验证 Release 构建）。

### CMake

- 最低 3.16  
- 若使用 **CMake 4.x**，配置 PortAudio 子项目时需增加：

  ```text
  -DCMAKE_POLICY_VERSION_MINIMUM=3.5
  ```

  否则 PortAudio 19.7.0 的 `cmake_minimum_required(2.8)` 会导致配置失败。

### 运行时（播放测试）

- 已安装 **Dante Virtual Soundcard**（或其它 ASIO 驱动），且能在其它 ASIO 应用中见到设备。

## 2. 获取源码与 SDK

确保以下路径有效（或准备在 CMake 中自定义）：

| 变量 | 默认路径 |
|------|----------|
| `PORTAUDIO_ROOT` | `E:/opensource/portaudio/portaudio-19.7.0` |
| `ASIOSDK_ROOT_DIR` | `E:/opensource/portaudio/ASIO-SDK_2.3.4_2025-10-15/ASIOSDK` |

验证 ASIO SDK：

```text
%ASIOSDK_ROOT_DIR%\common\asio.h   必须存在
```

## 3. 配置与编译

### 命令行（推荐）

```powershell
cd E:\opensource\portaudio\PortAudio-ASIO

cmake -S . -B build `
  -G "Visual Studio 18 2026" -A x64 `
  "-DCMAKE_POLICY_VERSION_MINIMUM=3.5"

cmake --build build --config Release
```

输出：`build\Release\LtcAsioPlayer.exe`

### 自定义依赖路径

```powershell
cmake -S . -B build `
  -DPORTAUDIO_ROOT="D:/libs/portaudio-19.7.0" `
  -DASIOSDK_ROOT_DIR="D:/libs/ASIOSDK" `
  -G "Visual Studio 17 2022" -A x64 `
  "-DCMAKE_POLICY_VERSION_MINIMUM=3.5"
```

### Debug 构建

```powershell
cmake --build build --config Debug
```

## 4. CMake 关键选项说明

根 `CMakeLists.txt` 在 `add_subdirectory(portaudio)` 前强制：

| 选项 | 值 | 含义 |
|------|-----|------|
| `PA_BUILD_STATIC` | ON | 静态链接 PortAudio |
| `PA_BUILD_SHARED` | OFF | 不生成 DLL |
| `PA_USE_ASIO` | ON | 启用 ASIO |
| `PA_USE_DS` / `WMME` / `WASAPI` / `WDMKS` | OFF | 仅 ASIO，减少干扰 |

应用目标：

| 设置 | 说明 |
|------|------|
| `project(... LANGUAGES C CXX)` | **必须包含 C**，否则 PortAudio 的 `.c` 源不会参与编译 |
| `VS_GLOBAL_UseOfMfc` = `Dynamic` | 链接共享 MFC，提供 `wWinMain` 入口 |
| `_AFXDLL` | 与共享 MFC 匹配 |
| `PA_USE_ASIO=1` | 应用侧可见 ASIO 扩展 API |

## 5. 常见构建问题

### ASIO SDK 未找到

**现象**：配置阶段报 `ASIOSDK_ROOT_DIR not found`。

**处理**：`-DASIOSDK_ROOT_DIR=` 指向包含 `common/asio.h` 的 **ASIOSDK** 目录，不是外层压缩包根目录。

### MFC 相关编译错误

**现象**：`Building MFC application with /MD requires MFC shared dll` 或缺少 `WinMain`。

**处理**：

- 确认已安装 MFC 组件  
- 确认 CMake 中 `VS_GLOBAL_UseOfMfc` 为 `Dynamic` 且定义了 `_AFXDLL`  
- 不要使用 `/MT` 静态 CRT 与共享 MFC 混用（本工程已对齐为 `/MD`）

### 链接时大量 `Pa_*` 未解析

**现象**：`portaudio_static` 中只有 ASIO 相关 `.obj`。

**原因**：父工程未启用 **C 语言**，`.c` 文件被标为“不参与编译”。

**处理**：`project(LtcAsioPlayer LANGUAGES C CXX)`（当前 CMakeLists 已包含）。

### 生成器不匹配

**现象**：切换 VS 版本后 CMake 报 generator 不一致。

**处理**：删除 `build` 目录后重新 `cmake -S . -B build ...`。

## 6. 在 Visual Studio 中打开

配置完成后可用 VS 打开：

```text
build\LtcAsioPlayer.sln
```

或 CMake 生成的 `.slnx`（视 VS 版本而定）。将 `LtcAsioPlayer` 设为启动项目，选择 `Release | x64` 编译运行。
