# 使用说明

## 1. 启动程序

运行 `LtcAsioPlayer.exe`（Release 默认路径：`build\Release\LtcAsioPlayer.exe`）。

启动后程序会：

1. 调用 `Pa_Initialize()` 初始化 PortAudio（仅 ASIO 后端）  
2. 枚举所有 **具备输出通道** 的 ASIO 设备并填入下拉框  
3. 若存在名称包含 **「Dante Virtual Soundcard」**（不区分大小写）的设备，则自动选中该项  
4. 根据当前设备刷新 **缓冲区** 列表与 **输出通道** 范围  

若初始化失败，日志区会显示 `Pa_GetErrorText` 返回的错误信息，相关控件会被禁用。

## 2. 界面控件说明

| 控件 | 说明 |
|------|------|
| **ASIO Device** | 下拉选择 ASIO 输出设备；切换后自动刷新缓冲区和通道范围 |
| **Output Channel** | 单声道 WAV 时，LTC 信号映射到的 ASIO 输出通道索引（从 0 开始）；立体声时固定使用 0、1，此项仅影响单声道 |
| **Buffer (frames)** | ASIO 缓冲区大小（采样帧数）；列表项中标注 `(preferred)` 的为驱动推荐值 |
| **Loop playback** | 勾选后播放到文件末尾自动从头循环；不勾选则播完自动停止 |
| **WAV File** | 显示当前文件路径；**Browse...** 打开文件对话框（`*.wav`） |
| **Play** | 加载 WAV 并打开 ASIO 流开始播放；播放中禁用设备/文件等设置 |
| **Stop** | 立即停止并关闭流 |
| **ASIO Control Panel** | 调用 `PaAsio_ShowControlPanel`，打开当前设备的驱动设置界面 |
| **Log** | 多行只读日志：初始化、采样率警告、播放参数、错误信息 |

## 3. 推荐 LTC 素材格式

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 采样率 | **48000 Hz** | 与 Dante Virtual Soundcard 常见工作速率一致 |
| 声道 | **单声道** | LTC 时间码通常为单通道音频 |
| 位深 | 16 / 24 bit PCM 均可 | 程序内部统一转为 32-bit float 输出 |
| 容器 | 标准 RIFF WAVE | 须含 `fmt` 与 `data` chunk |

支持的 `wFormatTag`：

- `1` — PCM  
- `3` — IEEE float（32-bit）

**不支持**：压缩格式（如 MP3 包装的 WAV）、多声道（>2）、非标准 chunk 排列异常的损坏文件。

## 4. 播放流程

```
选择设备 → 设置通道与缓冲区 → 选择 WAV → [可选] 循环 → Play
```

播放开始后日志示例：

```text
Playing: D:\audio\ltc.wav | 1 ch | 48000 Hz | buffer 512 | out ch 0 | loop off
```

### 采样率不一致

若 WAV 采样率与设备 `defaultSampleRate` 整数部分不同，日志会给出 **Warning**，但仍以 **WAV 采样率** 调用 `Pa_OpenStream`。

若打开流失败：

1. 点击 **ASIO Control Panel**，在 Dante 驱动中将采样率设为与 WAV 一致（如 48000 Hz）  
2. 尝试其它缓冲区大小  
3. 确认无其它程序独占 ASIO 设备  

## 5. LTC 信号路由（Dante）

### 单声道 LTC

- 应用向 PortAudio 提交 **1 个逻辑输出声道**  
- 通过 `PaAsioStreamInfo.channelSelectors` 映射到所选 **Output Channel**（默认 **0** = Dante 第 1 路输出）  

### 立体声 WAV

- 应用提交 **2 个逻辑声道**，分别映射到 ASIO 通道 **0** 与 **1**  
- UI 中的 Output Channel 不改变立体声映射  

### Dante Controller

本程序只负责向 ASIO 驱动送音频；**Dante 网络路由、订阅关系** 须在 **Dante Controller** 或 Virtual Soundcard 配置中单独设置，并确认对应通道有电平变化。

## 6. 停止与退出

- **Stop**：随时停止；可再次 **Play** 同一文件  
- 自然播完（未勾选循环）：日志显示 `Playback finished.`，控件恢复  
- 关闭对话框：自动 `Pa_StopStream` / `Pa_Terminate`，释放资源  

## 7. 典型使用场景

1. **向 Dante 网络发送 LTC**：本机播放 LTC WAV → Virtual Soundcard 某输出 → Dante 路由至远端设备  
2. **本地验证 LTC 文件**：监听指定 ASIO 通道是否有正确波形（需外接监听或软件电平表）  
3. **调试缓冲区与延迟**：更换 Buffer 帧数，观察打开流是否稳定、延迟变化
