# 项目概述

## 项目名称

**LTC ASIO 播放器**（`LtcAsioPlayer`）

## 项目目标

在 Windows 上实现一款 MFC 对话框应用，通过 **PortAudio 的 ASIO Host API** 将预先录制的 **LTC 时间码 WAV**（或其它 PCM WAV）播放到 **Dante Virtual Soundcard** 等 ASIO 声卡的指定输出通道，供下游设备或软件接收时间码信号。

## 技术栈

| 层次 | 技术 | 版本 / 说明 |
|------|------|-------------|
| 语言 | C++11 | 应用逻辑；PortAudio 核心为 C |
| UI | MFC | 共享 MFC DLL（`UseOfMfc=Dynamic`、`_AFXDLL`） |
| 音频 I/O | PortAudio | 19.7.0，静态库，仅启用 ASIO |
| 驱动接口 | Steinberg ASIO SDK | 2.3.4，源码编入 `portaudio_static` |
| 构建 | CMake | 3.16+；CMake 4.x 需策略最低版本 3.5 |

## 设计原则

- **不直接调用 ASIO SDK 播放**：播放路径统一走 PortAudio API（`Pa_OpenStream` 等），ASIO SDK 仅作为 PortAudio 编译依赖。
- **UI 与音频解耦**：`AudioEngine`、`DeviceEnumerator`、`WavFileReader` 不依赖 MFC，便于维护与测试。
- **实时安全**：PortAudio 回调中只做内存拷贝与原子计数，不在回调里分配/释放内存或调用 UI。

## 音频路由策略

| WAV 声道数 | PortAudio 输出 | ASIO 物理通道 |
|------------|----------------|---------------|
| 单声道（LTC 常见） | `channelCount = 1` | 映射到 UI 所选通道（默认 **0**，即第 1 路） |
| 立体声 | `channelCount = 2` | 固定映射到设备通道 **0、1** |

## 功能范围（已实现）

- ASIO 输出设备枚举与下拉选择  
- 默认选中名称包含 `Dante Virtual Soundcard` 的设备  
- 查询并选择 ASIO 合法缓冲区大小（含 preferred 标记）  
- 输出通道号（Spin + Edit）  
- WAV 文件浏览与加载  
- 播放 / 停止 / 循环播放  
- 日志区与 `Pa_GetErrorText` 错误提示  
- `PaAsio_ShowControlPanel` 打开驱动控制面板  

## 不在本阶段范围

- 实时 SMPTE/LTC 解码与时间码显示（仅回放 PCM 文件）  
- 直接使用 ASIO SDK 编写独立播放逻辑  
- macOS / Linux 等非 Windows 平台  
- 网络流、录音、多文件队列等高级功能  

## 默认依赖目录

与 `CMakeLists.txt` 中 `CACHE PATH` 一致：

```
E:/opensource/portaudio/portaudio-19.7.0
E:/opensource/portaudio/ASIO-SDK_2.3.4_2025-10-15/ASIOSDK
```

ASIO SDK 根目录下须存在 `common/asio.h`（即 `ASIOSDK` 子目录，而非仅外层 `ASIO-SDK_*` 文件夹）。
